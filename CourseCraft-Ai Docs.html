CourseCraft AI
Platform Overview

An AI-powered academic program design platform that automates curriculum generation, course design, outcome mapping, and program scheduling using Groq LLM.

React + Vite frontend
Node.js + Express backend
Groq LLM (llama-3.3-70b)
MongoDB Atlas
Core features
What CourseCraft AI does
Eight integrated features covering the full academic content design workflow.

Curriculum Builder

Generates a complete semester-wise academic curriculum from program inputs. Produces course codes, credit distribution, prerequisites, elective slots, and program outcomes.

Groq LLM Pre-computed credit plan OBE principles

Course Generation

Creates detailed course syllabi with unit-wise breakdowns, topics, estimated hours, course objectives, prerequisites, and optional lab experiment plans.

Groq LLM From curriculum or manual Lab syllabus tab

Outcome Mapping

Generates course outcomes tagged to Bloom's taxonomy levels (Remember → Create). Optionally produces a CO-PO correlation matrix. Accepts syllabus via textbox or PDF upload.

Groq LLM Bloom's taxonomy PDF extraction CO-PO matrix optional

Program Generator

Generates week-wise schedules for workshops, bootcamps, and short courses. Supports beginner/intermediate/advanced calibration and optional capstone project integration.

Groq LLM 1–50 weeks Capstone option

Contextual Chatbot

Chat interface that answers questions about any selected curriculum, course, or program using the JSON data as context. Out-of-context questions return a styled red error.

Groq LLM JSON context injection Multi-turn history

Dashboard & Activity

Central dashboard showing stats (curricula, courses, outcomes, programs), quick action cards, and a unified recent activity feed across all content types with click-to-view.

React Query Real-time stats Unified activity feed

Profile & Personalization

User profile management with name update, password change, profile reset (deletes all content), and custom AI instructions that are appended to every Groq generation request.

JWT auth Custom AI instructions Data reset

Export & Download

Generates formatted PDF and DOCX reports from stored MongoDB JSON data using pdfkit and the docx npm package. Files are generated on demand and served directly for download.

pdfkit docx npm Direct download

Implementation
How each feature is implemented
Key implementation decisions for each major feature.
1

Curriculum Builder — credit-safe generation

Credits are never generated by Groq. JavaScript pre-computes exact per-semester credit targets using buildSemesterPlan(), distributes them into course slots using assignCreditsToSlots(), then merges the JS-computed credits into Groq's output after parsing. Groq only generates course names, types, prerequisites, and difficulty levels.

JS builds plan → Groq generates names only → merge credits in JS → validate → save
2

Prerequisite ordering — post-processing validator

After merging credits, a JS pass builds a codeToSemester map and checks every prerequisite. If a prerequisite points to the same or a later semester, it is automatically cleared to null. The prompt also enforces ordering rules to reduce occurrences.

Build codeToSemester map → for each course: if prereq.semester >= course.semester → set null
3

Course generation — two-source approach

Users can generate a course from an existing curriculum (auto-fills course fields from the selected curriculum's course list) or enter details manually. The Groq prompt generates unit-wise syllabus with topics and hours. If includesLab is true, a separate lab syllabus tab is rendered in the result.

Option 1: select curriculum → pick course → auto-fill → edit → generate Option 2: manual entry → fill all fields → generate
4

PDF extraction — client-side only, never stored

pdfjs-dist runs entirely in the browser. The PDF is read as an ArrayBuffer, text is extracted page by page, cleaned, and pasted into the syllabus textarea. The raw PDF file never reaches the server. Only the extracted text string is stored in MongoDB on the OutcomeMapping document.

FileReader → ArrayBuffer → pdfjs getDocument → getTextContent → clean → textarea PDF discarded · only text string stored in MongoDB
5

Outcome mapping — sequential two-step Groq pipeline

Step 1 generates course outcomes with Bloom's taxonomy tags. Step 2 uses those COs as input to generate the CO-PO matrix. Both run in sequence inside a single controller function. The CO-PO matrix step is optional — triggered only when the user checks the matrix checkbox. bloomsUtils.js normalizes any "ing" verbs to present tense before display.

generateCourseOutcomes() → [if checkbox ticked] generateCOPOMatrix(COs) → save both
6

Program Generator — difficulty and capstone calibration

The Groq system prompt includes explicit week-count calibration (1–2 weeks = crash course, 7–12 = bootcamp, 25–50 = full course) and difficulty-level tone instructions. When includesCapstone is true, the last 2–3 weeks are reserved for capstone work and a capstoneProject object is generated and validated separately.

inputs → difficulty rules + week targets injected → Groq → validate week count → if capstone: verify last week title contains "Capstone" → auto-fix if not
7

Chatbot — JSON context injection

On each chat request, the selected content's full JSON (curriculum, course, or program) is serialized into a structured plain-text context string and injected into the Groq system prompt. Groq is instructed to return the literal string "OUT_OF_CONTEXT" for unrelated questions. The backend checks for this string and sets isOutOfContext: true. The frontend renders this in a red error bubble. The last 10 messages of conversation history are sent on each turn for multi-turn context.

contextData → buildContextSummary() → system prompt → last 10 messages → Groq response === "OUT_OF_CONTEXT" → isOutOfContext: true → red bubble in UI
8

Authentication — JWT with bcrypt

Passwords are hashed with bcrypt (10 salt rounds) via a Mongoose pre-save hook. Login returns a JWT signed with JWT_SECRET, stored in localStorage as cc_token. All protected routes use authMiddleware which verifies the token and attaches req.user. An axios response interceptor clears the token and redirects to /login on any 401.

register → hash password → save → sign JWT → return token login → find user → comparePassword → sign JWT → return token
AI pipeline
Groq LLM — five generation functions
All AI calls use llama-3.3-70b-versatile on Groq with response_format: json_object to enforce structured output.
Function 1

generateCurriculum()

Generates course names, types, difficulty, and prerequisites only. Credits injected from JS. Temperature: 0.3. Max tokens: 8000.

Function 2

generateCourseSyllabus()

Generates unit titles, topics, hours, objectives, prerequisites, and lab experiments. Temperature: 0.4. Max tokens: 6000.

Function 3

generateCourseOutcomes()

Generates 5–8 COs with Bloom's level, verb, and category. Distribution percentages must sum to 100. Temperature: 0.3. Max tokens: 4000.

Function 4

generateCOPOMatrix()

Maps each CO to each PO using scale 0–3. Conservative by design — most cells should be 0 or 1. Temperature: 0.2. Max tokens: 3000.

Function 5

generateWeeklyProgram()

Generates week-wise schedule with topics, activities, deliverables. Week count and totals computed in JS post-parse. Temperature: 0.5. Max tokens: 8000.

Function 6

chatWithContext()

Injects JSON context as system prompt. Returns OUT_OF_CONTEXT for unrelated queries. Maintains last 10 messages for multi-turn. Temperature: 0.5. Max tokens: 1024.

All functions use callWithRetry(fn, retries=3, delay=2000ms) with exponential backoff Rate limit (429) errors and validation errors trigger automatic retry response_format: { type: "json_object" } enforced on all generation calls Computed values (credits, totals, distributions) always recalculated in JS after parsing
